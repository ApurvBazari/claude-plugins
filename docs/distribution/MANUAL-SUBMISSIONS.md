| Other License | _(leave blank)_ |

**Description:**
```
Analyzes existing codebases and generates tailored Claude Code tooling — CLAUDE.md files, path-scoped rules, project-specific skills and agents, hook configurations, and CI/CD workflows — derived from the actual languages, frameworks, and enforced configs detected in the codebase rather than generic templates. Includes a drift-detection mode that suggests updates as the codebase evolves, and a plugin-aware generation mode that skips capabilities already covered by installed plugins.
```

**Validate Claims:**
```
Install the marketplace (claude marketplace add https://github.com/ApurvBazari/claude-plugins), install the plugin, and run /onboard:init in any medium-size existing repo. The wizard completes in 3–5 minutes and produces: a root CLAUDE.md with build/test/lint commands, path-scoped rules derived from the detected ESLint/Prettier/tsconfig (not generic templates), a project-specific code-reviewer agent, and auto-format + lint hooks wired to the detected tooling. All generated artifacts are inspectable as plain files under .claude/ before anything is committed; /onboard:evolve later diffs the current codebase against the generated tooling and suggests drift-based updates.
```

**Specific Task(s):**
```
Clone any medium-size TypeScript/Next.js project that has ESLint and Prettier configured (e.g., a typical app generated by create-next-app). In that directory, install the plugin and run /onboard:init.
```

**Specific Prompt(s):**
```
Run /onboard:init. When the wizard asks for a preset, select "Standard". Let it finish. Then ask Claude: "Summarize the CLAUDE.md files, rules, skills, and hooks you just generated. Show me the path-scoped rules that were derived from my actual lint/format configs."
```

**Additional Comments** _(optional, include this)_:
```
Part of the apurvbazari-plugins marketplace (three plugins, MIT licensed). onboard is the flagship — companion plugins (notify for hooks, handoff for session continuity) are submitted separately in subsequent issues, one at a time, per the "no other open issues" rule.
```

### Issue 2 — notify

**⚠️ Submit only after Issue 1 (onboard) is closed or merged.**

**Title:**
```
[Resource]: notify — Cross-platform system notifications for Claude Code
```

**Field values:**

| Field | Value |
|---|---|
| Display Name | `notify` |
| Category | `Hooks` |
| Sub-Category | `General` |
| Primary Link | `https://github.com/ApurvBazari/claude-plugins` |
| Author Name | `Apurv Bazari` |
| Author Link | `https://github.com/ApurvBazari` |
| License | `MIT` |
| Other License | _(leave blank)_ |

**Description:**
```
Cross-platform system notifications for Claude Code hooks, supporting macOS via terminal-notifier and Linux via notify-send. Duration filtering suppresses notifications for short-lived events, git context (repo and branch) is extracted automatically, and contextual messages are generated based on the triggering hook type.
```

**Validate Claims:**
```
Install the plugin and run /notify:setup. Select the stop and notification events, set minDurationSeconds to a reasonable threshold (e.g., 10 seconds). Trigger a long-running task in Claude Code — a native system notification appears on completion. Trigger a short task — no notification (filtered). Works out of the box on macOS with terminal-notifier installed; on Linux with notify-send (libnotify). Missing backends fail silently per hook-safety rules. Config lives at notify-config.json and is editable without restart.
```

**Specific Task(s):**
```
Install terminal-notifier (macOS: brew install terminal-notifier) or ensure notify-send is available (Linux). Install the plugin and run /notify:setup with minDurationSeconds set to 10.
```

**Specific Prompt(s):**
```
After setup, ask Claude: "Please run a test suite or other long-running command in the shell and let me know when it's done." Observe the system notification that fires on completion, and note that short commands (under 10s) do not notify.
```

**Additional Comments** _(optional, include this)_:
```
Hook script always exits 0 — never blocks Claude. No network requests. Companion to onboard (codebase analysis) and forge (scaffolder) in the same marketplace: https://github.com/ApurvBazari/claude-plugins
```
