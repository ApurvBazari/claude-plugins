01 — what it does

From day-one setup to continuous upkeep

A guided /onboard:start analyzes your codebase, asks a short adaptive set of questions, and generates a complete, tailored Claude Code tooling setup — root and subdirectory CLAUDE.md, path-scoped rules, project-specific skills and agents, hooks, output styles, and metadata.

That part is well-covered — Claude Code's /init, Copilot, Codex, Cursor, and several web tools all generate a CLAUDE.md. onboard's focus is the second half: the artifacts it writes carry self-maintaining headers and, in enriched mode, auto-evolution hooks that watch your dependency manifests and structure. As the code drifts away from the config, /onboard:evolve reads that accumulated drift and proposes targeted realignments you approve one by one.

The focus

Generation is well-covered. Maintenance is the harder half. The /onboard:evolve drift-detection loop keeps your AI configs aligned with the code that actually exists, not the code that existed on the day you ran setup.

02 — the drift loop

The drift-detection loop

When /onboard:start runs in enriched mode it installs quiet auto-evolution hooks. They record what changed; /onboard:evolve turns that record into a reviewed set of config updates.

The categories /onboard:evolve recognises: new languages and dependencies, structural shifts (a new workspace package appearing), config diffs (tsconfig.json paths), and missing or stale hooks and rules. You decide which proposed updates land; the snapshot then updates so the next run is incremental.

03 — architecture

The four-phase generation pipeline

/onboard:start runs a fixed pipeline: an empty-repo guard, a read-only analysis pass, an adaptive wizard, generation through a dedicated agent, then an education handoff. Click a phase for detail.

Three specialised components do the work, each clickable above: the read-only codebase-analyzer agent scans structure, dependencies and patterns; the wizard skill tailors its questions to what the analyzer found; the config-generator agent takes the analysis plus wizard answers and writes every artifact. The analysis report stays in conversation context — it is never written to disk.

05 — v2 cutover

onboard 2.0: the hard v1 → v2 cutover

Version 2.0.0 moves the headless contract to a new v2 context shape (version: 2, phases.*, syntheses, dependencies). The cutover is hard and intentional — there is no migration helper.

Behaviouronboard 1.10.0 (v1)onboard 2.0.0 (v2)
Headless context shapev1 flat contextversion: 2 (phases / syntheses)
v1 input to generate✓ accepted— rejected outright
Migration helper— n/a— none ships
Where v1 callers stayonboard@1.10.0pin to 1.10.0
CI/CD & sprint-contract rendering— external↑ rendered from cicdAndDelivery
Root v2 schema stability—✓ stable (minors fill deferred phases)

Why a clean break

The generate skill detects the version at Step 0 and refuses v1 input — never silent, never partial, with a machine-parseable error so callers can route. v1 callers must stay on onboard@1.10.0 for the life of their session.

07 — output

What it generates

Core artifacts are always emitted. Enriched artifacts land when the wizard or a headless flag enables them — including the auto-evolution hooks that power the drift loop.

core — always
CLAUDE.md
Root + subdirectory

Project overview, stack, commands, conventions, and context-specific guidance for major directories.

.claude/rules/*.md
Path-scoped rules

Testing, APIs, components, security — scoped to the directories they govern.

.claude/skills + agents
Project tooling

Stack-specific skills and specialised agents — plugin-aware, skipping anything an installed plugin already covers.

.claude/settings.json
Hook entries

Auto-format and lint checks tailored to your tooling, plus a PR template and commit conventions.

output-styles + meta
Style + manifest

A project-scoped output style tuned to your archetype, and .claude/onboard-meta.json as the generation manifest.

enriched — when enabled
CI/CD pipelines
GitHub Actions

ci, tooling-audit, and pr-review workflows rendered from the cicdAndDelivery synthesis.

auto-evolution hooks
FileChanged + SessionStart

Plus drift-detection scripts — the machinery that feeds /onboard:evolve.

harness artifacts
progress + guide

docs/progress.md and docs/HARNESS-GUIDE.md for cross-session tracking.

sprint contracts
docs/sprint-contracts/

JSON exit criteria, plus agent-team support (quality hooks, env vars).

Every generated file carries a self-maintaining header (version + date) that prompts Claude to notify you when the configuration drifts from actual code patterns. Supported stacks: Node.js / TypeScript, Python, Go, Rust, Java / Kotlin, Ruby, monorepos (npm / yarn / pnpm workspaces, Turborepo, Nx, Lerna), and mixed-language projects.
