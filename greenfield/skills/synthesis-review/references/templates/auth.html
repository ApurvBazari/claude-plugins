8. Downstream impact
Auth decisions cascade forward to three downstream phases:
  • Privacy — consent flows, right-to-erasure, and data subject requests depend on auth lifecycle (deletion flow) and tenant resolution (which data belongs to whom).
  • Security — threat model depends on auth strategy, IdPs, MFA posture, and service-to-service auth surface. Security hardening builds on the auth baseline.
  • Runtime Operations — session store sizing (Redis for cookie-sessions at scale), token refresh rate (impacts API gateway throughput), and audit log volume (impacts storage budget).
Approve or adjust auth before walking those phases — misalignment here will force synthesis re-runs downstream.
