Domain mode governs which DDD constructs are captured. full-ddd = entities + bounded contexts + aggregates + value objects + domain events + ubiquitous language + ACLs. ddd-lite drops value objects, domain events, and ACLs. light collapses to a single default bounded context and skips aggregate distinction entirely.

ACLs prevent external vocabulary from leaking into the domain model. Each ACL recorded here identifies an external system and the mapping layer (e.g., "legacy CRM uses Lead/Account — map to Customer/Order at boundary"). The apiIntegration phase must implement an adapter or translation layer for each ACL. Missing ACLs when external systems are listed in apiIntegration fire a contradiction in grill-spec.
