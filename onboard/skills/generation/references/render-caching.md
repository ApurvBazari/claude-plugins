# Render module: `phases.caching` → caching infrastructure

## When to render

- `phases.caching.skipped == true` → SKIP this module (no-op).
- `phases.caching.layers == []` → SKIP (nothing to wire).
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `lib/cache.ts` | always (when not skipped) | TS client wrapping the chosen cache layers |
| `next.config.ts` patch | `phases.P2.stack.framework == "next"` AND `layers[]` includes `"cdn"` or `"isr"` | header rules + `revalidate` defaults |
| `lib/cache/.gitkeep` | always | placeholder for layer-specific helpers |

## next.config.ts patch (Next.js + CDN/ISR layers)

```typescript
// Generated by onboard render-caching.md from phases.caching
const config = {
  experimental: {
    staleTimes: {
      dynamic: ${phases.caching.ttlDefaults.dynamic ?? 0},
      static: ${phases.caching.ttlDefaults.static ?? 300},
    },
  },
  async headers() {
    return [
      {
        source: "/api/:path*",
        headers: [
          { key: "Cache-Control", value: "s-maxage=${phases.caching.ttlDefaults.api ?? 60}, stale-while-revalidate" },
        ],
      },
    ];
  },
};
export default config;
```

## Backward compatibility

- `phases.caching.skipped == true` → no files written; no error.
- `phases.caching.layers[] == []` → same as skipped.
- If `next.config.ts` already exists, emit `next.config.caching.patch.ts` alongside it and surface a merge note in the generation report rather than clobbering user config.
