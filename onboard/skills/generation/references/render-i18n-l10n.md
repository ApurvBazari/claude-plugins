# Render module: `phases.i18nL10n` → i18n / l10n wiring

## When to render

- `phases.i18nL10n.skipped == true` → SKIP this module (no-op).
- `phases.i18nL10n.targetLocales` is empty or absent → SKIP (no locales to wire).
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `lib/i18n.ts` | always (when not skipped) | i18n library init for the chosen `library` |
| `messages/<locale>.json` | one per locale in `targetLocales[]` | default locale gets seed strings; others get skeletons |
| `next.config.ts` patch | `routingStrategy` ∈ `{"sub-path", "domain"}` AND framework is Next.js | `i18n: { locales, defaultLocale }` merge |
| `app/[locale]/layout.tsx` | `delivery == "lazy"` AND `routingStrategy == "sub-path"` | locale-scoped layout for dynamic routing |

`next.config.ts` is **merge-only**: read existing config, append the `i18n` key, never overwrite siblings. `messages/<locale>.json` collisions surface a merge note and write as `messages/<locale>.generated.json` alongside.

## lib/i18n.ts template

```typescript
// i18n init for ${phases.i18nL10n.library}
// Locales: ${phases.i18nL10n.targetLocales.join(", ")}
// Default: ${phases.i18nL10n.defaultLocale}
// Delivery: ${phases.i18nL10n.delivery}  // "eager" | "lazy"
// Generated by onboard render-i18n-l10n.md

// next-intl      → `import { getRequestConfig } from "next-intl/server"; export default getRequestConfig(async ({ locale }) => ({ messages: (await import(\`../messages/\${locale}.json\`)).default }));`
// react-i18next  → `import i18n from "i18next"; import { initReactI18next } from "react-i18next"; i18n.use(initReactI18next).init({ lng: "${defaultLocale}", resources: { /* lazy-loaded */ } });`
// formatjs       → `import { createIntl, createIntlCache } from "@formatjs/intl"; export const cache = createIntlCache(); export function getIntl(locale: string, messages: Record<string, string>) { return createIntl({ locale, messages }, cache); }`
// lingui         → `import { i18n } from "@lingui/core"; export async function activate(locale: string) { const { messages } = await import(\`../messages/\${locale}.json\`); i18n.load(locale, messages); i18n.activate(locale); }`
// native-intl    → `export function format(locale: string, value: number) { return new Intl.NumberFormat(locale).format(value); }`

export {};
```

Emit only the line that matches `phases.i18nL10n.library`. When `library == "native-intl"`, skip the `messages/*.json` skeletons (Intl APIs don't need a message catalog) — write only `lib/i18n.ts` plus optional routing patches.

## messages/<locale>.json template

For the default locale:

```json
{
  "common.appName": "${phases.i18nL10n.appNameHint || 'App'}",
  "common.loading": "Loading…",
  "common.error": "Something went wrong"
}
```

For every non-default locale in `targetLocales[]`, write a skeleton with the same keys and empty-string values:

```json
{
  "common.appName": "",
  "common.loading": "",
  "common.error": ""
}
```

The skeletons are a translation-team handoff signal — the wizard chose the locale list, not the copy.

## next.config.ts patch (App-Router and Pages-Router)

When `routingStrategy ∈ {"sub-path", "domain"}` AND the framework is Next.js, merge into the existing `nextConfig` object:

```typescript
// Merged by onboard render-i18n-l10n.md
const nextConfig = {
  // ...existing keys preserved...
  i18n: {
    locales: ${JSON.stringify(phases.i18nL10n.targetLocales)},
    defaultLocale: "${phases.i18nL10n.defaultLocale}",
    // localeDetection: false  // uncomment if routingStrategy == "domain"
  },
};
```

For `routingStrategy == "domain"`: emit `localeDetection: false` and a TODO comment pointing to the `domains[]` config the developer must fill in (the wizard captures locale list, not domain assignment).

## app/[locale]/layout.tsx template (lazy delivery + App Router)

```tsx
// Generated by onboard render-i18n-l10n.md
// Library: ${phases.i18nL10n.library}
import type { ReactNode } from "react";

export function generateStaticParams() {
  return ${JSON.stringify(phases.i18nL10n.targetLocales.map(l => ({ locale: l })))};
}

export default function LocaleLayout({
  children,
  params: { locale },
}: { children: ReactNode; params: { locale: string } }) {
  return <div data-locale={locale}>{children}</div>;
}
```

When `library == "next-intl"`: wrap children in `<NextIntlClientProvider locale={locale} messages={...}>`. When `library == "react-i18next"`: call `activate(locale)` server-side before render.

## Backward compatibility

- `phases.i18nL10n.skipped == true` → no files written; no error.
- `phases.i18nL10n.targetLocales == []` → same as skipped.
- If `lib/i18n.ts` already exists, emit alongside as `lib/i18n.generated.ts` and surface a merge note in the report.
- `next.config.ts` is never overwritten — only the `i18n` key is merged in; existing keys win.
- Pre-R6 contexts with no `phases.i18nL10n` key produce zero artifacts (silent skip).
