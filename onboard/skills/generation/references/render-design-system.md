# Render module: `phases.designSystem` → design system wiring

## When to render

- `phases.designSystem.skipped == true` → SKIP this module (no-op).
- `phases.designSystem.componentLibrary == "none"` AND `storybookAdopted != true` AND all of `typographyScale`/`colorSystem`/`spacingTokens` are empty strings → SKIP.
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `components.json` | `componentLibrary == "shadcn"` | shadcn CLI config (style + tailwind path + aliases) |
| `app/layout.tsx` patch | `componentLibrary == "mui"` | wrap `<body>` in `<ThemeProvider theme={theme}>` + emotion cache |
| `app/layout.tsx` patch | `componentLibrary == "mantine"` | wrap `<body>` in `<MantineProvider>` |
| `tailwind.config.ts` patch | `componentLibrary == "mui"` OR any token field non-empty | merge typography / color / spacing tokens into `theme.extend` |
| `.storybook/main.ts` | `storybookAdopted == true` | Storybook stories glob + framework |
| `.storybook/preview.ts` | `storybookAdopted == true` | global decorators + `parameters` with brand tokens |

`app/layout.tsx` and `tailwind.config.ts` are **merge-only** — never clobber existing exports.

## app/layout.tsx patch (MUI)

```tsx
// Generated by onboard render-design-system.md
import { AppRouterCacheProvider } from "@mui/material-nextjs/v14-appRouter";
import { ThemeProvider } from "@mui/material/styles";
import CssBaseline from "@mui/material/CssBaseline";
import { theme } from "@/lib/theme";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <AppRouterCacheProvider>
          <ThemeProvider theme={theme}>
            <CssBaseline />
            {children}
          </ThemeProvider>
        </AppRouterCacheProvider>
      </body>
    </html>
  );
}
```

When `componentLibrary == "mantine"`: replace the MUI imports with `MantineProvider` from `@mantine/core` and wrap with `<MantineProvider defaultColorScheme="light">`. Cross-reference `phases.designSystem.themingApproach` to choose between `css-variables` and `tailwind-config` injection styles.

## tailwind.config.ts patch (tokens)

```typescript
// Generated by onboard render-design-system.md
import type { Config } from "tailwindcss";

const config: Config = {
  content: ["./app/**/*.{ts,tsx}", "./components/**/*.{ts,tsx}"],
  theme: {
    extend: {
      // typographyScale — emit when phases.designSystem.typographyScale is non-empty
      fontSize: {
        // e.g. { xs: "0.75rem", sm: "0.875rem", base: "1rem", lg: "1.125rem", xl: "1.25rem" }
      },
      // colorSystem — emit when phases.designSystem.colorSystem is non-empty
      colors: {
        // e.g. { brand: { 50: "#…", 500: "#…", 900: "#…" } }
      },
      // spacingTokens — emit when phases.designSystem.spacingTokens is non-empty
      spacing: {
        // e.g. { 18: "4.5rem", 22: "5.5rem" }
      },
    },
  },
  plugins: [],
};

export default config;
```

The wizard captures `typographyScale` / `colorSystem` / `spacingTokens` as free-form strings (designer-authored token snippets). Inline them verbatim under `theme.extend` — do not attempt to parse them into structured objects.

## Backward compatibility

- `phases.designSystem.skipped == true` → no files written; no error.
- `componentLibrary == "none"` with empty tokens AND `storybookAdopted == false` → same as skipped.
- `app/layout.tsx` / `tailwind.config.ts` already exist → emit `.designsystem.patch.{ts,tsx}` siblings and surface a merge note rather than clobbering.
- `.storybook/*` files already exist → preserve existing files; emit `.storybook/<name>.generated.ts` siblings instead.
