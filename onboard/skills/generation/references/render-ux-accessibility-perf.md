# Render module: `phases.uxAccessibilityPerf` → UX / a11y / perf + 3 inline gates

## When to render

- `phases.uxAccessibilityPerf.skipped == true` → SKIP this module (no-op).
- All of `performanceBudgets`, `imageOptimization`, `fontLoading`, and `concerns.*` are empty/`"none"`/`needed: false` → SKIP.
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `.github/workflows/lighthouse-ci.yml` | `phases.cicdAndDelivery.draftProvider == "github-actions"` (or absent → default GHA) | Lighthouse CI workflow keyed to PRs + main |
| `lighthouse.config.js` | `performanceBudgets.{lcp,inp,cls}` populated | CWV budget JSON encoded into Lighthouse assertions |
| `next.config.ts` patch | `imageOptimization != "none"` AND framework is Next.js | `images.{remotePatterns, formats, deviceSizes}` |
| `app/layout.tsx` font snippet | `fontLoading != "none"` AND `routingStrategy == "app-router"` | `next/font` import + `<html className={font.variable}>` |
| `lib/email/marketing-${vendor}.ts` | `concerns.marketingEmail.needed == true` | marketing-email client init for the chosen vendor |
| `lib/push/${vendor}.ts` | `concerns.pushNotifications.needed == true` | push-notification client init for the chosen vendor |
| `lib/analytics/${vendor}.ts` | `concerns.productAnalytics.needed == true` | product-analytics client init for the chosen vendor |

`<vendor>` is `concerns.<gate>.vendor` lowercased and slugified (e.g., `"Mailchimp"` → `mailchimp`, `"OneSignal"` → `onesignal`, `"PostHog"` → `posthog`). The `next.config.ts` and `app/layout.tsx` patches are **merge-only** — never clobber existing exports.

## .github/workflows/lighthouse-ci.yml template

```yaml
# Generated by onboard render-ux-accessibility-perf.md
name: Lighthouse CI
on:
  pull_request:
  push:
    branches: [main]
jobs:
  lighthouse:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npm run build
      - name: Run Lighthouse CI
        run: |
          npm install -g @lhci/cli@0.13.x
          lhci autorun --config=./lighthouse.config.js
```

When `phases.cicdAndDelivery.draftProvider` is `"gitlab-ci"` / `"circleci"` / `"none"` → skip this file (R6 ships GHA only; non-GHA providers in a later round).

## next.config.ts image-optimization patch

```typescript
// Generated by onboard render-ux-accessibility-perf.md
const nextConfig = {
  images: {
    // imageOptimization === "next-image" → leave defaults; just declare remotePatterns
    // imageOptimization === "imgix" / "cloudinary" → set loader: "custom" with loaderFile
    formats: ["image/avif", "image/webp"],
    deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048, 3840],
    remotePatterns: [],
  },
};
export default nextConfig;
```

When `imageOptimization == "imgix"` or `"cloudinary"`: additionally emit `lib/image-loader.ts` referenced from `loaderFile`.

## Backward compatibility

- `phases.uxAccessibilityPerf.skipped == true` → no files written; no error.
- Each inline gate is independently gated by `concerns.<gate>.needed == true` AND a non-empty `vendor`. Missing vendor → skip that gate's lib file and surface a soft warning.
- Lighthouse + image-optimization + font snippets all gate on their own enum values — emit only the ones whose source field is populated.
- If any target path already exists, emit `.generated.{ts,tsx,js,yml}` alongside and surface a merge note rather than clobbering.
