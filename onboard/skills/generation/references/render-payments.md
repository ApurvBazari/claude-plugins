# Render module: `phases.payments` → payments infrastructure

## When to render

- `phases.payments.skipped == true` → SKIP this module (no-op).
- `phases.payments.provider == "none"` → SKIP.
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `lib/payments/<provider>.ts` | always (when not skipped) | provider SDK init + checkout/subscription/portal helpers |
| `app/api/webhooks/<provider>/route.ts` | always | signed webhook handler for the provider |
| `app/(payments)/portal/page.tsx` | `phases.payments.customerPortal == true` | self-serve customer portal entry page |
| `.env.example` | always — merged, not clobbered | provider-specific keys appended |

`<provider>` is `phases.payments.provider` lowercased (e.g., `stripe`, `paddle`, `lemonsqueezy`, `polar`, `razorpay`).

## app/(payments)/portal/page.tsx (when `customerPortal == true`)

```tsx
// Self-serve customer portal — generated by onboard render-payments.md
import { createPortalSession } from "@/lib/payments/${phases.payments.provider}";
import { redirect } from "next/navigation";

export default async function PortalPage() {
  // Resolve the current customer from auth; mint a portal session and redirect.
  const session = await createPortalSession("CUSTOMER_ID", "/account");
  redirect(session.url);
}
```

## .env.example additions (merge, do not clobber)

Append (idempotently) to the existing `.env.example`:

```dotenv
# Payments — ${phases.payments.provider}
${phases.payments.provider.toUpperCase()}_API_KEY=
${phases.payments.provider.toUpperCase()}_WEBHOOK_SECRET=
# Optional: ${phases.payments.provider.toUpperCase()}_PUBLISHABLE_KEY=
```

For Stripe also append `STRIPE_PRICE_ID_*` placeholders matching `phases.payments.priceCatalog[]` if present.

## Backward compatibility

- `phases.payments.skipped == true` → no files written; no error.
- `phases.payments.provider == "none"` → same as skipped.
- `.env.example` is merge-only: never overwrite existing keys; append missing ones with empty values.
- If the webhook route or portal page already exists, emit alongside as `.generated.{ts,tsx}` and surface a merge note.
