# Render module: `phases.realtime` → realtime infrastructure

## When to render

- `phases.realtime.skipped == true` → SKIP this module (no-op).
- `phases.realtime.transport == "none"` → SKIP.
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `lib/realtime.ts` | always (when not skipped) | TS client + channel/event types |
| `app/api/realtime/route.ts` | Next.js App Router | server endpoint (SSE / WS upgrade / broker auth) |
| `pages/api/realtime.ts` | Next.js Pages Router | same as above, Pages-style handler |
| `lib/realtime/reconnect.ts` | always | reconnect/backoff helper aligned with `reconnectStrategy` |

Choose App Router vs Pages Router from `phases.P2.stack.routerStyle` (default: App Router). Emit exactly one server endpoint file.

## lib/realtime/reconnect.ts

```typescript
// Reconnect strategy: ${phases.realtime.reconnectStrategy}
export function makeReconnector(connect: () => Promise<void>) {
  let attempt = 0;
  return async function reconnect() {
    const delay = Math.min(30_000, 2 ** attempt++ * 250);
    await new Promise((r) => setTimeout(r, delay));
    await connect();
  };
}
```

## Backward compatibility

- `phases.realtime.skipped == true` → no files written; no error.
- `phases.realtime.transport == "none"` → same as skipped.
- If a server endpoint already exists at the target path, emit alongside as `.generated.ts` and surface a merge note in the report.
