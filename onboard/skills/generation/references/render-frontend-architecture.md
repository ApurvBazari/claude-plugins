# Render module: `phases.frontendArchitecture` → frontend architecture skeletons

## When to render

- `phases.frontendArchitecture.skipped == true` → SKIP this module (no-op).
- All three of `stateManagement`, `dataFetching`, `errorBoundaries` equal `"none"` → SKIP (nothing to wire).
- Else → render the artifacts below.

## Output paths

| File | Condition | Content |
|---|---|---|
| `package.json` deps patch | always (when not skipped) | runtime + dev deps for the chosen libs |
| `lib/store.ts` | `stateManagement` ∉ `{"builtin-only", "none"}` | typed store skeleton for the chosen library |
| `lib/queries.ts` | `dataFetching` ∉ `{"fetch", "none"}` | data-fetching client init (QueryClient / SWRConfig / Apollo / etc.) |
| `app/error.tsx` | `errorBoundaries != "none"` AND `routingStrategy == "app-router"` | Next.js App Router error boundary |
| `pages/_error.tsx` | `errorBoundaries != "none"` AND `routingStrategy == "pages-router"` | Pages-router error boundary |

`package.json` is **merge-only**: append missing deps to `dependencies` / `devDependencies` and never overwrite existing pins.

## app/error.tsx template (App Router)

```tsx
// Generated by onboard render-frontend-architecture.md
// Strategy: ${phases.frontendArchitecture.errorBoundaries}  // "per-route" | "per-feature" | "global-only"
"use client";

export default function Error({ error, reset }: { error: Error; reset: () => void }) {
  return (
    <main role="alert">
      <h1>Something went wrong</h1>
      <p>{error.message}</p>
      <button onClick={reset}>Try again</button>
    </main>
  );
}
```

For `errorBoundaries == "per-feature"`: additionally emit a `.gitkeep` under `app/(features)/.error-boundary-here` so the developer remembers to colocate per-feature `error.tsx` files.

## Backward compatibility

- `phases.frontendArchitecture.skipped == true` → no files written; no error.
- All three lib outputs are independently gated — emit only the ones whose enum is non-`"none"`.
- If `lib/store.ts` or `lib/queries.ts` already exists, emit alongside as `lib/store.generated.ts` / `lib/queries.generated.ts` and surface a merge note in the report.
- `package.json` is never overwritten — only missing keys are appended; existing pins win.
